Network Slicing

Category: 5G / core
Executive Summary:
Network Slicing logically divides a single physical 5G infrastructure into multiple, isolated virtual networks. Each slice is mathematically optimized for a specific use case, such as autonomous driving or massive IoT.

Advanced Technical Definition

Relying on SDN and NFV, this architecture logically segregates the gNB, transport networks, and the Next Generation Core (NGC).

Slice TypePrimary TargetTypical Application
eMBB20 Gbps ThroughputUHD Video, AR/VR
URLLC< 1 ms LatencyV2X, Smart Factories
mMTCHigh Density / Low PowerMassive IoT, Sensors


For a URLLC slice, the network instantiates the UPF and SMF locally at the Edge Cloud to guarantee sub-millisecond response times, while standard eMBB traffic routes normally through the AMF.
